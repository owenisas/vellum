# API supplement

OpenAPI is auto-generated by FastAPI at `/docs` and `/openapi.json`. This document covers the
auth requirements and a few example flows.

## Auth requirement matrix

| Endpoint | Auth | Permission |
|---|---|---|
| `GET /api/health` | Public | — |
| `GET /api/models` | Public | — |
| `POST /api/chat` | Required | `chat:invoke` |
| `POST /api/detect` | Public | — |
| `POST /api/strip` | Required | any |
| `POST /api/apply` | Required | any |
| `POST /api/anchor` | Required | `anchor:create` |
| `POST /api/verify` | Public | — |
| `POST /api/proof/text` | Public | — |
| `GET /api/proof/tx/{tx_hash}` | Public | — |
| `GET /api/proof/spec` | Public | — |
| `POST /api/companies` | Required | `company:create` (or `admin_secret` in demo mode) |
| `GET /api/companies` | Required | any |
| `GET /api/chain/*` | Public | — |
| `GET /api/solana/*` | Public | — |
| `GET /api/demo/scenario` | Public | — |
| `POST /api/demo/reset` | Required | `admin:reset` |
| `GET /api/responses` | Public | — |
| `GET /api/responses/latest` | Public | — |

When `AUTH0_DOMAIN` is empty, the server returns a `DEMO_IDENTITY` carrying every scope, so all
endpoints work without an Authorization header (good for local development).

## Example: anchor a piece of text

```bash
# 1. Create a company (auto-generates a keypair)
COMPANY=$(curl -s -X POST http://localhost:5050/api/companies \
  -H 'Content-Type: application/json' \
  -d '{"name": "Acme", "auto_generate": true, "admin_secret": "dev-admin-secret"}')
ISSUER_ID=$(echo "$COMPANY" | jq -r '.issuer_id')
PRIV=$(echo "$COMPANY" | jq -r '.private_key_hex')

# 2. Generate watermarked text (fixture mode → deterministic)
CHAT=$(curl -s -X POST http://localhost:5050/api/chat \
  -H 'Content-Type: application/json' \
  -d '{"messages": [{"role": "user", "content": "demo"}]}')
TEXT=$(echo "$CHAT" | jq -r '.text')

# 3. Sign client-side (any EIP-191 personal_sign tool — eth-account, ethers, etc.)
HASH=$(printf "%s" "$TEXT" | shasum -a 256 | awk '{print $1}')
# (Use a small Python helper to sign; not shown.)

# 4. Anchor
curl -s -X POST http://localhost:5050/api/anchor \
  -H 'Content-Type: application/json' \
  -d "{\"text\": \"$TEXT\", \"signature_hex\": \"$SIG\", \"issuer_id\": $ISSUER_ID}"
```

## Verifying the proof bundle

```bash
curl -s -X POST http://localhost:5050/api/verify \
  -H 'Content-Type: application/json' \
  -d "{\"text\": \"$TEXT\"}"
```

The response includes `proof_bundle_v2`, which carries an `anchors[]` entry plus
`verification_hints` (RPC URLs for Solana, explorer link).
